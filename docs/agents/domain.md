# Domain docs

Read `GLOSSARY.md` for durable domain vocabulary, project-specific terminology, mode names, controller concepts, and canonical language.

`GLOSSARY.md` is not:

- a PRD
- a task list
- a current implementation plan
- an issue tracker
- a scratchpad
- a changelog

For feature intent, use the active PRD GitHub issue.
For implementation work, use the linked vertical-slice GitHub issue.
For current behaviour, verify against source code and tests.
For durable architecture decisions, use `docs/adr/`.
For external API/reference material, use `docs/reference/`.
