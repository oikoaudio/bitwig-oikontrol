# Reference material

`docs/reference/` describes external API docs and reference projects used to verify behaviour.

Reference material is a verification source, not a feature requirement. Feature requirements come from approved PRD parent issues and linked implementation issues.

The existing `ref/bitwig-api-flat/` directory remains the local Bitwig API reference and should be checked before guessing about API surface or controller-script behaviour.

Other local reference copies under `ref/` may include controller scripts, manuals, and related projects. Treat them as examples to verify against, not as automatic design authority for this repository.
