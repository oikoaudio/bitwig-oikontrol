# handoff

## Purpose

Produce compact re-entry notes after partial or completed work.

Prefer GitHub issue or PR comments for issue-specific handoffs. Use `.scratch/handoffs/` for local work, exploratory tasks, or handoffs not yet tied to a durable tracker item.

## When To Use

Use this skill when work is stopping, changing owners, waiting for review, or needs a concise state snapshot.

## Inputs To Inspect

- Current issue, PRD, or task
- Changed files and diff
- Check results
- Open questions and blockers
- Relevant `.scratch/` notes for the active workflow only

## Process

1. Identify the current state in one or two sentences.
2. List changed files and what changed.
3. Record checks run and results.
4. Record checks not run and why.
5. Note risks, caveats, blockers, and assumptions.
6. Recommend the next concrete step.

## Output Format

Use:

- Current state
- Changed files
- Checks run
- Checks not run
- Risks/caveats
- Blockers
- Next recommended step

## Stop Conditions

Stop after producing the handoff.

Do not continue implementation, create PRDs, draft issues, or start parallel agents unless explicitly instructed.
